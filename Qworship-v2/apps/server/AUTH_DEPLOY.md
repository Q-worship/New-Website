# Auth API deployment (marketing sign-up / OTP)

Deploy the **auth-only** Express server so the marketing site Worker can proxy `/api/auth/*` via `AUTH_API_ORIGIN`.

## Quick deploy (Render)

1. Create a [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) cluster (free tier is fine).
2. Create a [Resend](https://resend.com) API key and verify a sending domain (or use `onboarding@resend.dev` for testing).
3. In [Render](https://render.com) → **New** → **Blueprint** → connect this GitHub repo.
4. Point at [`render.yaml`](render.yaml) (repo path: `Qworship-v2/apps/server/render.yaml` or root blueprint if copied).
5. Set secrets when prompted:
   - `MONGODB_URI` — Atlas connection string
   - `RESEND_API_KEY` — from Resend
   - `JWT_SECRET` — auto-generated by Render, or set manually
6. After deploy, note the service URL (default name: `https://qworship-auth-api.onrender.com`).

## Smoke test

```bash
curl -i -X POST https://qworship-auth-api.onrender.com/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"firstName":"Test","lastName":"User","email":"you@example.com","password":"TestPass123!"}'
```

Expected: **201** with `{"success":true,"nextStep":"/verify"}`.

Health check: `GET https://qworship-auth-api.onrender.com/health` → `{"ok":true}`.

## Wire marketing site

1. Set `AUTH_API_ORIGIN` on the **new-website** Cloudflare Worker to your deployed URL (no trailing slash).
   - Already in [`wrangler.jsonc`](../../../wrangler.jsonc) as `https://qworship-auth-api.onrender.com` after redeploy.
2. Redeploy site: `npm run deploy:site` from repo root.

## Docker (any host)

```bash
cd Qworship-v2/apps/server
docker build -f Dockerfile.auth -t qworship-auth-api .
docker run -p 5000:5000 \
  -e NODE_ENV=production \
  -e MONGODB_URI="..." \
  -e JWT_SECRET="..." \
  -e RESEND_API_KEY="..." \
  -e EMAIL_FROM="Qworship <verify@qworship.com>" \
  qworship-auth-api
```

## Environment variables

| Variable | Required | Notes |
|----------|----------|--------|
| `MONGODB_URI` | Yes | Same DB as main app if sharing users |
| `JWT_SECRET` | Yes | Long random string |
| `NODE_ENV` | Yes | `production` (enables Resend email) |
| `RESEND_API_KEY` | Yes (prod) | OTP delivery |
| `EMAIL_FROM` | Yes (prod) | Verified sender in Resend |
| `PORT` | No | Default `5000` |
| `FRONTEND_URL` | No | CORS; proxy makes this optional for marketing site |

## Do not use `app.qworship.com` unless that host runs this auth code

If `POST /api/auth/signup` on `AUTH_API_ORIGIN` returns **405**, the origin does not implement the auth API — deploy here instead.
